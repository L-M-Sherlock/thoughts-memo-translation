# 如何从对话中学习

[Mini Blog Post 5: How to Learn From Conversations — Neel Nanda](https://www.neelnanda.io/blog/mini-blog-post-5-how-to-learn-from-conversations)

6月14日

[Neel Nanda]著(https://www.neelnanda.io/?author=5ed927247689ec0247c03233)

我目前正参加一个[虚拟会议](https://www.eaglobal.org/events/eagxvirtual2020/)，所以我脑子里一直思考的一件事就是：如何从那些比我涉猎更广的人身上获得最大的收获。如何尽可能多地获取信息，如何高效获取，以及如何确保我真正理解并记住了这些信息！我认为这是一种非常可以习得的技能，也是我在过去 3 年里重点培养的一项技能。与有趣的人进行迷人的对话是我人生的一大乐事，我非常看重能否把这件事做好！在这篇文章中，我将尝试分享一些我开发的核心工具！

健康警告：我收到的反馈是，我在谈话中表现得相当激进，有时会给人一种我在评判对方是否值得我浪费时间的既视感。我努力减少这些情况，并关注对方的舒适度，但这似乎是这种方法难以避免的代价！

- **提问**

- 这一点不言自明——多提问！你要让对方感到越轻松越好。引导一场对话、弄清楚该聊**什么**需要大量的认知劳动，而提问能让对方的生活轻松得多。

- **技巧：留意好奇心**

- 我发现对话（尤其是闲聊）往往最后都在聊一些并没那么有趣的事情。这太糟糕了！如果你聊的是你们双方都感到兴奋的事情，那是双赢！

- 我对此的主要对策是，留意对方所说内容中透出的那一点点好奇或兴奋的苗头。通常一个人的陈述中包含很多不同的观点和信息，其中一些比其他的更令人兴奋。我发现练习内省自己对每一部分的兴奋程度，并针对最酷的那部分提问，是非常有价值的。

- 这通常需要几轮迭代，但它是从闲聊转向新鲜酷事的一种非常有效的方法！

- 内省是一项困难的技能，通常「留意情绪」是我可以[长篇大论](https://www.lesswrong.com/posts/GLPaZamxqkx7XJbXv/the-skill-of-noticing-emotions)的话题。但这是你可以通过练习变得更好的事情。

- **要点**：你想在情绪上获得尽可能多的**接触面（surface area）**。留意你何时感受到它、共同的趋势、相关的身体感觉。向你的大脑发出信号，说明这很重要，对你很有意义。

- 在脑海中模拟历史案例并解构好奇的感觉，有助于在未来察觉到它。

- **技巧：留意困惑！**

- 如果他们提到了某些感觉有点不对劲或古怪的地方，问清楚！

- 准确地组织一个问题通常很难。在那些情况下，只需说「你提到了 ___，我对于为什么那样可行感到有点困惑，你能详细说明一下吗？」就非常有用了！当我对自己困惑的点感到困惑时，这比尝试提出一个具体的问题更有效。

- **变体**：留意当他们提到一个对我来说非常陌生的世界维度时。

- **提示**：「关于那个话题，我能聊过 30 秒吗？」如果不能，就问！

- 内在体验是，我正试图在脑海中为对方所说的话构建一个模型。每隔一段时间，我就会注意到模型中有一个漏洞，或者它报错了。**那**就是困惑的感觉。

- **增加对话的「接触面」**

- 提问的瓶颈通常在于缺乏背景，不知道该问什么！当我与一个完全陌生领域的专家交谈时，这非常糟糕——我缺乏直觉来知道该问什么，或者什么是有趣的。

- 我把这种背景加直觉称为**接触面（surface area）**，我认为一个核心目标是弄清楚如何尽可能多地获取它。

- **技巧：利用他们的直觉**

- 方便的是，你正在和一个拥有大量接触面的人交谈，你通常可以通过正确的问题来获取这些！

- 例如：「当你进入 X 领域时，什么让你感到惊讶」，「人们经常误解什么」，「你所在领域最重要的问题是什么」。

- **技巧：索要案例**

- 案例是学习事物的一种**绝佳**方式！对话常常会陷入抽象术语和概念的漫谈中，从而失去与我实际理解内容的联系。如果我能理解一个案例，那通常能传递大量信息。

- 拥有一个具体的案例通常能激发问题，指出困惑或好奇点，因为即便我不理解整体构想，我也能**理解**一个案例。

- 从信息论的角度来看：我们**想要**交流概念，但我们必须将它们转化为语言。这可能会损失大量信息。案例是另一种沟通渠道，可以识别翻译中的错误。

- 我建立的一个**核心**心理习惯是：每当脑海中感到困惑时，总是索要一个案例。

- **技巧：转述**

- 转述意味着用**我自己的话**重复一遍我**认为**他们正在表达的意思，并询问是否正确。

- **非常重要**：始终用**询问**而非告知。保持谦逊，误解是常有的事！如果你歪曲了他们的观点，很容易显得讨厌、傲慢或是在曲解原意。

- 这是一项**极其**核心的技能。在大多数对话中，我每隔 5 分钟左右就会做一次，这能识别出大量的误解。

- 如果你能从这篇文章中学到一件事，那就是转述！

- 转述是我主要的报错工具，因为通过将内容转化回我的语言，我经常能发现错误。对方纠正错误，比他们从头开始清晰地解释某事要容易得多。

- 如果你持怀疑态度，我高度建议你试一试。第一次听错的次数多得令人不安。

- 这也有助于我留存信息，它给了我处理的时间，并迫使我检查自己是否**真的**听懂了。

- 在谈话中走马观花、偶尔走神是很简单的。转述迫使我保持活跃并真正参与其中。

- **另一个有趣的加成**：我认为将复杂想法转化为文字的能力是非常有用的。转述的习惯迫使我进行**大量的**常规练习，在过去的几年里，我这项技能变得显著增强了。

- 这明显提高了我的学习和教学能力。
