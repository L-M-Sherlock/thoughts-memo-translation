# 集中处理低强度任务

原文：[Batch low-intensity tasks together | What's new](https://terrytao.wordpress.com/career-advice/batch-low-intensity-tasks-together/)

> **尝试涉猎诸多领域，远比将所有精力集中于一件事要容易得多。** ([昆提利安](http://en.wikipedia.org/wiki/Quintilian))

工作中面对的任务，大致可以分为两类：

1. 「高强度」任务：这类任务较为复杂，需要全神贯注、集中精力（例如撰写研究论文；备课或准备学术报告；起草一封冗长、详尽且严谨的电子邮件；思考数学问题；阅读研究论文或专著），以及

2. 「低强度」任务：这类任务属于例行事务（但可能相当费时），不必消耗太多心力（例如填写各类表格；进行已经提前准备妥当的授课或学术报告；简要回复邮件；办事与赴约；查阅邮件或浏览网页）。

处理高强度任务所需的「思维模式」，与处理低强度任务相当不同。（以我自己的体会为例，要完全专注于一个数学问题，将所有相关背景知识融会贯通、了然于胸，往往需要持续思考至少半小时。）为减少在不同「思维模式」间来回切换所带来的精神疲劳，我发现，把相似的低强度任务集中处理，并在时间上（或空间上）将它们与高强度任务区隔开来，会很有帮助。

例如，你可以专门辟出一整块时间，将堆积在案头的诸多「琐碎」事务一扫而光。把所有「容易让人分心」的任务（如办公室答疑时间和其他预约）安排在同一天；尽量把授课的日子集中起来；凡此种种。（举例而言，我曾发现，相比在相同的日子教两门不同的课，或在截然不同的时间教同一门课，将一门微积分大课的两个教学班背靠背讲授（比如早上九点一次，十点紧接着再上一次），能显著节省备课时间以及心神。）

这种分块安排不仅能在时间上进行，也能在空间上实现。例如，在管理纸质文件方面，我在办公室里划出了一小块区域，专门放将来很可能还要再次处理的「重要」表格和记录；而在办公室另一处更大的区域，放那些以后再用到的概率较低（但并非完全没有）的「普通」文书。由于我很少需要从后一类文件中找东西，所以不会花太多精力去整理或归档；它们往往会堆成一座小山，我每隔几个月左右再整理一次（并且大多直接会丢掉）。（不过，我确实会尽量让「重要」表格保持相对有序，也避免让海量的「普通」表格把它们弄得杂乱。）

同理，可以准备一个「待寄」文件架，专门用来存放低优先级的纸质信件，攒齐后一次性全部寄出，而不必一天往返收发室好几趟。（当然，趁出门寄信的机会，正好顺道查看一下自己的信箱，或处理其他需要在系里走动的杂务。）

类似地，我发现保留一份「五分钟任务」的常设清单很有帮助。我把它们写在黑板上，这类任务优先级低，也不需要花费太多时间和脑力（尽管严格来说，有些或许超出了五分钟的界限），适合在日程安排中有短暂空闲、且精力和动力一般的时候去完成。这包括：检查不那么紧急、无需时刻盯着的备用邮箱账户、文件夹或实体信箱；更新自己的简历、个人网页或社交媒体资料；整理办公室、电脑、手机或在线效率工具；或者处理琐碎的事务性工作，诸如安排会议、上传表格。我发现，一旦备好这份常设清单，还可当作「杂物堆」。有时突然想起一些小任务，迟早需要完成，却出于种种原因不便立刻着手，就先记在上面。等到终于有合适的空闲时，再从清单上逐一划掉，就我个人而言，这是一种很有满足感的体验。

至于电子邮件，流水线作业法似乎非常高效：先积攒起来，然后（a）过第一遍：删掉垃圾邮件；（b）再过一遍：处理那些容易应对的邮件（只需读一次即可删除或归档的，只需三言两语就能回复的，或是可以移入某个「待处理」文件夹的）；然后（c）如果你觉得时机合适，再处理一封或多封需要长篇回复的邮件。

如果你习惯在一天中浏览多个网站，我建议使用[内容聚合器](http://en.wikipedia.org/wiki/Aggregator)（我曾用 [Google Reader](http://www.google.com/reader/view/) 来做这件事，直至它停止服务），这样你就可以将浏览活动集中起来一次完成，避免它们不时分散你在其他事务上的注意力。

如果手头有一批低强度、低优先级的任务，那不妨先将其搁置，等到真正被高强度的工作耗得需要喘口气时，再处理。比如，当我在研究中遇到瓶颈，感到特别沮丧时，我就觉得这是个好时机，可以去办攒下来的杂务，处理积压的文书工作，哪怕只是补看邮件、浏览网页也好。身边备着这类轻松的任务，正好可以在创造力恢复之前，以一种还算有成效的方式打发时间。

需要注意的是，批量处理只适用于**低强度**任务。高强度任务需要高度专注，会大幅消耗精神耐力，如果将此类活动与其他低强度或高强度任务混在一起，可能会适得其反，或让人分心。
